# 序数论之类的东西

序数理论与下列领域密切相关：

1. **集合论核心结构**
   - 基本概念：基数（cardinal）、良序（well-ordering）
   - 构造：超限递归、传递集、等级结构 $V_\alpha$
   - 应用：ZFC 公理系统中的归纳与定义原则
2. **证明论与逻辑**
   - 证明论序数（proof-theoretic ordinal）：衡量形式系统强度，如
      $\text{PA}\to\varepsilon_0,\ \text{ACA}_0\to\Gamma_0$。
   - 超限归纳与可定义性分析（反映计算可终止性）。
3. **描述集合论与递归论**
   - Borel、projective 层级以序数表示复杂度。
   - 可计算序数（Church-Kleene $\omega_1^{CK}$）。
4. **拓扑与序结构**
   - 序数空间 $[0,\alpha)$ 的拓扑性质（如紧致性、第一可数性）。
   - 序类型分类与序数拓扑。
5. **组合与大基数理论**
   - 序数分划（partition calculus）、不动点、大基数定义。
6. **计算机科学和类型理论**
   - 递归式终止证明使用序数作为秩函数。
   - λ-演算与类型系统的归纳定义依赖超限序。

tmux 是一个**终端复用器（terminal multiplexer）**，用于在同一个终端窗口中创建、管理和保持多个会话、窗口与分屏布局。它常与 Vim/Neovim、SSH、Alacritty、Linux/WSL 等组合使用，是高效命令行环境的核心工具之一。

**性质**

- 类型：终端复用器
- 平台：Linux / macOS / BSD / WSL（Windows Terminal、Alacritty、Git Bash 都可使用）
- 核心作用：管理会话、窗口、分屏；断线保持；长任务后台运行

**核心特性**

1. **会话保持（Session Persistence）**
   - 离开服务器或关闭终端，tmux 会话仍继续运行
   - 重新连接后任务仍在
   - 非常适合 SSH 环境、长时间训练模型、跑脚本
2. **窗口与分屏管理**
   - 同一 tmux 会话中可以有多个窗口（类似标签页）
   - 每个窗口可以分成多个 vertical/horizontal pane
   - 所有 pane 都在同一终端进程里管理
3. **快捷键操作**
   - 默认前缀是 `Ctrl + b`
   - 分屏、切换、调整大小、创建窗口全部用快捷键完成
   - 可自定义前缀为 `Ctrl + a`（Emacs、tmux 经典习惯）
4. **脚本化与自动化**
   - tmux 可以自动创建会话、窗口、pane
   - 多任务工作流可完全由脚本描述
   - 适合重复性任务或开发环境初始化
5. **SSH 场景优势巨大**
   - 断网、断电、切换网络不会杀死进程
   - 远程服务器上的长任务安全运行
6. **插件生态（基于 TPM）**
   - 透明状态栏
   - 电池/时间/Git 状态显示
   - 辅助导航
   - 配色主题
   - 与 Neovim、fzf 集成

**优点**

- 会话不会掉，非常稳定
- 实现真正的多窗口、多分屏
- 跨平台（尤其适合 WSL/Linux）
- 高度定制化、脚本化
- 适用于远程服务器

**缺点**

- 初始学习曲线较高
- 默认快捷键不够直观（建议改前缀）
- 使用鼠标需要开启相关选项
- Windows 原生环境（非 WSL）支持有限

**常见工作流**
1）本地 Linux/macOS/WSL：

- `tmux new -s work`
- 分屏：`Ctrl+b %` / `Ctrl+b "`
- 切 pane：`Ctrl+b 箭头`
- 脱离会话：`Ctrl+b d`
- 恢复会话：`tmux attach -t work`

2）远程 SSH：

- SSH 登录远程服务器
- 启动 tmux，会话保持
- 网络中断后重新 ssh -> tmux attach

3）配合 Vim/Neovim：

- 左窗编辑，右窗跑脚本、测试
- 通过快捷键移动 pane，比 Alt+Tab 快

**适用人群**

- SSH 用户
- Vim/Neovim 用户
- Linux/WSL 开发者
- 需要后台跑长任务的人
- 想提高终端效率的用户

**与 Windows Terminal/Alacritty 的关系**

- tmux 是运行在终端中的“窗口管理器”
- Windows Terminal/Alacritty 是外层的终端模拟器
- 二者不冲突，可叠加使用

忘记sudo密码怎么办

###### ChatGPT said:

WSL 中忘记 sudo 密码可以在 **Windows 主机端** 直接重置，无需知道原密码。方法如下。

**步骤 1：以管理员身份打开 Windows PowerShell**
右键 PowerShell → 以管理员运行。

**步骤 2：进入你的 WSL 发行版设置，列出用户**
以 Ubuntu 为例：

```
ubuntu config --default-user root
```

或新版（WSL2 常见）：

```
wsl -d Ubuntu -u root
```

执行后会直接以 root 身份进入 WSL。

**步骤 3：在 WSL 内重置你的用户密码**
假设你的用户名是 `yourname`，执行：

```
passwd yourname
```

输入两次新密码即可。

**步骤 4：把默认用户改回去（可选但推荐）**
如果你用的是新版注册方式，退出 WSL 后在 PowerShell 输入：

```
ubuntu config --default-user yourname
```

或：

```
wsl -d Ubuntu --set-default-user yourname
```

**现在你就可以用新密码正常 sudo 了。**
